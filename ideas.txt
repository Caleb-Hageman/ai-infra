Feature 3: Fast Control Plane + Heavy Data Plane Split
------------------------------------------------------
Task 3.1: Split API responsibilities
- Subtask: Control plane handles auth, metadata, signed URL/session
  issuance, and status reads.
- Subtask: Data plane handles **AI chunking**, embedding, indexing, and
  storage writes (heavy path today; should not sit on upload HTTP thread
  for large inputs).
- Subtask: Return immediate job/session IDs for async progress tracking.

Task 3.2: Separate services by workload
- Subtask: Create/assign `chat-service` for low-latency request/response
  paths.
- Subtask: Create/assign `ingest-service` for long-running
  ingestion/indexing jobs.
- Subtask: Ensure independent scaling/operational ownership per service.

Feature 4: Async Ingestion Pipeline
-----------------------------------
Task 4.1: Queue-based ingestion
- Subtask: On upload complete (GCS object ready), publish ingestion job
  message.
- Subtask: Worker consumes message and runs **AI chunking (HF path) →
  embed → index** pipeline on the stored object.
- Subtask: Persist job status and error details for client polling.

Task 4.2: Event-driven processing
- Subtask: Use GCS object finalize event to trigger ingestion job
  creation.
- Subtask: Ensure idempotent event handling (same event does not
  double-process).
- Subtask: Add dead-letter/retry path for failed jobs.

Feature 5: Chat Runtime Execution Optimization
----------------------------------------------
Task 5.1: Reduce warm-path latency
- Subtask: Profile and time each stage (retrieval, rerank, model call,
  response formatting).
- Subtask: Parallelize independent I/O calls where safe.
- Subtask: Add lightweight caching for repeated retrieval/model
  sub-results where applicable.
